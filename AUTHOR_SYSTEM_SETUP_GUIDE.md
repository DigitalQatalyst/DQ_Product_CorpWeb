## Features

### Author Creation
- **Modal Interface**: Click "New Author" button to open creation modal
- **Required Fields**: Name, title, bio, avatar (URL or file upload)
- **Optional Fields**: LinkedIn, Twitter, website URLs, email
- **Avatar Upload**: Supports both URL input and file upload to Supabase Storage
- **Validation**: Real-time form validation with error messages

### Author Selection
- **Search Functionality**: Search authors by name or title
- **Visual Selection**: Author cards with avatar, name, and title
- **Auto-populate**: Selected author data automatically fills blog form fields
- **Create on Demand**: Create new authors without leaving the blog form

### Enhanced Blog Content Editor
- **Larger Editor**: Minimum 500px height for better writing experience
- **Enhanced Toolbar**: More formatting options (H1-H6, quotes, code blocks)
- **Content Preservation**: Proper HTML structure preservation
- **Auto-save Indication**: Visual feedback that content is being preserved
- **Word/Character Count**: Real-time statistics

## Usage Instructions

### Creating a New Author

1. **Navigate to Blog Creation**: Go to `/admin-ui/media/new?tab=Blog`
2. **Open Author Selection**: In the "Author Selection" section (blue box)
3. **Click "New Author"**: Opens the author creation modal
4. **Fill Required Fields**:
   - Full Name (e.g., "Dr. John Smith")
   - Professional Title (e.g., "AI Strategy Expert")
   - Biography (professional background)
   - Avatar (URL or upload file)
5. **Add Optional Social Links**: LinkedIn, Twitter, website
6. **Submit**: Author is created and automatically selected

### Selecting an Existing Author

1. **Search Authors**: Use the search box to find authors by name/title
2. **Browse Authors**: Scroll through the author list
3. **Click to Select**: Click on an author card to select them
4. **Auto-populate**: Form fields are automatically filled with author data

### Writing Blog Content

1. **Enhanced Editor**: Use the larger rich text editor with expanded toolbar
2. **Structure Content**: Use H1-H6 headings to structure your content
3. **Format Text**: Bold, italic, underline, strikethrough, quotes, code
4. **Add Lists**: Bullet points and numbered lists
5. **Insert Links**: Add hyperlinks with the link button
6. **Content Preservation**: All formatting is preserved exactly as entered

## Data Flow

### Author Data Structure
```typescript
interface Author {
  id: string;
  name: string;
  title: string;
  bio: string;
  avatar_url?: string;
  linkedin_url?: string;
  twitter_url?: string;
  website_url?: string;
  email?: string;
  is_active: boolean;
  created_at: string;
  updated_at: string;
}
```

### Blog Form Integration
When an author is selected:
- `formData.author` = author.name
- `formData.authorRole` = author.title  
- `formData.authorBio` = author.bio
- `formData.authorAvatar` = author.avatar_url
- `formData.authorLinkedIn` = author.linkedin_url

### Content Preservation
The enhanced editor saves content in multiple formats:
- `formData.bodyJson` = TipTap JSON structure (for editing)
- `formData.bodyHtml` = Clean HTML (for display)
- `formData.body` = HTML backup (for compatibility)

## Validation Rules

### Author Creation
- **Name**: Required, non-empty string
- **Title**: Required, non-empty string  
- **Bio**: Required, non-empty string
- **Avatar**: Required, either URL or uploaded file
- **Social URLs**: Optional, must be valid URLs if provided

### Blog Creation
- **Author Selection**: Must select an author (validates `selectedAuthor.id`)
- **Content**: Must have non-empty HTML content
- **Excerpt**: Required for hero section
- **Category**: Must select from predefined categories
- **Reading Time**: Must be positive number
- **Publication Date**: Required date
- **Hero Image**: Required, URL or uploaded file

## File Storage

### Author Avatars
- **Bucket**: `author-avatars` in Supabase Storage
- **Path**: `avatars/{authorId}-{timestamp}.{extension}`
- **Access**: Public read, authenticated write
- **Formats**: All image formats supported

### Blog Hero Images  
- **Bucket**: `media-thumbnails` in Supabase Storage
- **Path**: Auto-generated by media service
- **Access**: Public read, authenticated write
- **Recommended Size**: 1200x600px

## Troubleshooting

### Common Issues

1. **"Author avatars bucket not found"**
   - Run the storage setup SQL script
   - Check Supabase Storage permissions

2. **"Cannot upload avatar"**
   - Verify authentication status
   - Check file size (max 5MB recommended)
   - Ensure file is valid image format

3. **"Authors not loading"**
   - Check Supabase connection status
   - Verify RLS policies are enabled
   - Check browser console for errors

4. **"Content not saving properly"**
   - Check browser console for errors
   - Verify rich text editor is loading
   - Ensure form validation passes
